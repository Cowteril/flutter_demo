# Android Demo v0.1 特效升级计划

## 1. 背景

当前 Android Demo v0.1 已完成真实视频播放、沉浸式播放器、高光触发、双击爱心、6 类情绪粒子和分支选择反馈。但现有特效主要由 Flutter Icon、简单粒子和基础动画组成，视觉冲击力不足，难以在答辩录屏中形成明确记忆点。

本计划的目标是在不重构主播放器、不偏离 v0.1 范围的前提下，用有限时间把特效做出层次感和技术含量。

## 2. 时间约束

- 当前日期：2026-05-30
- 计划提交节点：2026-06-11
- 答辩节点：2026-06-19

因此本轮不追求完整商业级动效系统，而是优先完成可录屏、可解释、可验证的高价值特效升级。

## 3. 目标

### 3.1 产品目标

- 让用户在播放器中感知到明显的互动反馈增强。
- 让答辩录屏至少出现 1 个“招牌镜头”。
- 让当前 6 种情绪特效不再像临时 Demo，而像有设计方向的产品原型。

### 3.2 技术目标

- 使用 Flutter 生态中可解释、可落地的技术路线。
- 展示技术选型价值：代码动画、粒子系统、shader、交互动效。
- 保持 Android debug APK 可构建。
- 保持现有 v0.1 交互链路稳定。

### 3.3 合规目标

- 优先使用纯代码动效、Flutter shader、自制 Rive 或 AI 生成 sprite。
- CC0 素材库作为兜底，不作为第一选择。
- 新增外部下载素材时同步记录到 `docs/ASSET_LICENSES.md`。
- AI 生成素材只记录生成工具、用途和风格说明，不做版权限制讨论。

## 4. 非目标

- 不做 v0.2 的竖滑 Feed、情绪热力图、多指连击、手势施法、摇一摇和右侧操作栏。
- 不引入 Unity、Unreal 或原生 Android 特效引擎。
- 不大量堆 LottieFiles 免费素材作为最终方案。
- 不建立复杂资产审计流程。
- 不为了视觉效果牺牲播放器基础稳定性。

## 5. 核心策略

按“出现频率”和“视觉权重”分层处理特效。

| 层级 | 场景 | 技术策略 | 目标 |
| --- | --- | --- | --- |
| 高频低权重 | 双击爱心、普通点击反馈 | `flutter_animate` + 现有 EffectLayer | 轻量、不卡顿、反馈明确 |
| 中频中权重 | 爽、破防、撒糖、火焰、爱心等情绪粒子 | 自研粒子系统 + AI 生成 sprite + 少量 shader 闪白 | 明显比 Icon 粒子高级 |
| 低频高权重 | 分支选择、剧情爆发、高光时刻 | Fragment shader 或自制 Rive | 形成答辩录屏封面镜头 |

## 6. 技术选型

### 6.1 第一优先级：纯代码动效

建议新增：

- `flutter_animate`
- `confetti`

用途：

- 强化按钮入场、toast 弹出、选项卡片点击、爱心爆发节奏。
- 用 `confetti` 快速提升撒糖、爽点、成功反馈的粒子密度。
- 避免一开始就依赖外部素材。

说明：

- `flutter_animate` 是宽松许可证。
- `confetti` 是宽松许可证。
- 实际引入前仍以 `pub.dev` 当前 license 为准，并记录到实现说明中。

### 6.2 第二优先级：AI 生成 sprite 粒子

推荐来源：

- SDXL
- 即梦
- 其他可稳定产出透明 PNG 的 AI 生图工具

用途：

- 替换当前 `GenericParticleEffect` 中的 Material Icon / 几何形状。
- 统一生成 5-8 个同风格 sprite：
  - 光斑
  - 星星
  - 火焰
  - 水滴
  - 心形
  - 碎片
  - 糖果或圆形贴片

生成要求：

- 统一画风：霓虹、短剧爽感、透明背景、移动端高对比。
- 输出透明 PNG，尺寸建议 256x256 或 512x512。
- 同一批 prompt 生成，避免不同素材风格打架。
- 记录生成工具、生成日期、用途和核心 prompt，便于答辩说明 AI 辅助过程。

兜底来源：

- Kenney
- OpenGameArt 中明确标注 CC0 的素材

目录建议：

```text
client/assets/effects/particles/
  spark.png
  star.png
  flame.png
  droplet.png
  heart.png
  shard.png
```

### 6.3 第三优先级：Flutter Fragment Shader

用于做一个高技术含量的“剧情爆发”招牌镜头。

候选效果：

- 全屏冲击波
- 色散 Chromatic Aberration
- 扫描线
- 短暂闪白
- 径向扭曲
- 高光点触发时的视频层能量波

建议优先做：

```text
剧情高光爆发 = 闪白 + 径向冲击波 + RGB 色散 + 粒子爆裂
```

原因：

- 素材依赖为零。
- 技术含量明确，答辩容易解释。
- 和短剧“爽点/反转/战力爆发”强相关。

### 6.4 备选：自制 Rive 分支选择动画

用于做分支选择卡片的关键交互动效。

候选效果：

- 两条路线卡片浮起。
- 用户点击后路线发光。
- 选中路线扩散成路径线。
- 未选路线淡出。

Rive 更适合交互动效，但需要额外编辑器成本。若 shader 已经完成且时间充足，再做 Rive。

### 6.5 暂不作为主线：LottieFiles

Lottie 播放器本身可用，但风险在 JSON 动画素材许可证上。免费素材不能简单等同于可商用无风险。

策略：

- 可以用于临时原型。
- 最终提交版本只保留逐个核验过 license 的 Lottie。
- 关键动效优先用 shader、自制 Rive 或纯代码实现。

## 7. 落地阶段

### 阶段 1：代码动效增强

预计时间：0.5 天

任务：

- 引入 `flutter_animate` 和 `confetti`。
- 强化 `HeartBurstEffect`：
  - 中心爱心弹性缩放
  - 外围粒子分层扩散
  - 多次双击时错位叠加
- 强化 `TextFlyEffect`：
  - 入场弹性
  - 离场上浮
  - 轻微旋转或抖动
- 强化 `InteractionOverlay`：
  - 弹层入场动画
  - 选项按钮点击反馈

验收：

- 双击爱心观感明显优于当前版本。
- 普通选项反馈不影响播放控制。
- `flutter analyze` 和 `flutter test` 通过。

### 阶段 2：AI sprite 粒子替换

预计时间：0.5-1 天

任务：

- 生成 5-8 个统一风格 AI sprite。
- 如 AI 生成效果不稳定，再选择 CC0 sprite 兜底。
- 新建 `docs/ASSET_LICENSES.md`。
- 配置 `client/assets/effects/particles/`。
- 将 `GenericParticleEffect` 改为支持 sprite 粒子：
  - tears 使用水滴
  - candy 使用糖果/彩色圆片
  - flame 使用火焰
  - heart 使用心形
  - shockwave 叠加碎片或光斑

验收：

- 6 种情绪特效至少 4 种使用 sprite。
- AI 生成素材有生成说明；外部下载素材有 license 记录。
- APK 构建通过。

### 阶段 3：招牌 shader 镜头

预计时间：1-2 天

任务：

- 新增 shader 文件，例如：

```text
client/shaders/highlight_burst.frag
```

- 配置 `pubspec.yaml` shader assets。
- 新增 `HighlightBurstEffect`：
  - 高光触发或选项选择后播放
  - 叠加全屏 shader
  - 与现有 `EffectLayer` 生命周期统一
- 优先绑定到：
  - `EffectType.shockwave`
  - 剧情高光爆发
  - 战力爆发类 extension 高光

验收：

- 录屏中可明显看到全屏级爆发效果。
- 特效持续时间控制在 600-1200ms。
- 不阻塞点击、返回、播放控制。

### 阶段 4：分支选择关键动效

预计时间：0.5-1 天

两种实现路线：

方案 A：纯 Flutter 实现

- 选项卡片点击后：
  - 选中卡片放大
  - 边框流光
  - 路线 toast 弹出
  - 背景粒子扩散

方案 B：自制 Rive

- 使用 Rive 编辑器做一个小型状态机。
- Flutter 中按分支选择触发 state input。

建议：

- 若 shader 阶段按时完成，做方案 B。
- 若时间紧，做方案 A。

验收：

- 分支选择成为第二个可录屏重点。
- 分支反馈与普通情绪反馈仍互不污染。

### 阶段 5：音效补充

预计时间：0.5 天

任务：

- 选择 3-5 个音效：
  - tap
  - shockwave
  - branch_confirm
  - sweet/confetti
  - flame
- 优先来源：
  - Freesound CC0
  - Mixkit
  - Zapsplat，若使用则按要求署名
- 记录到 `docs/ASSET_LICENSES.md`。

验收：

- 关键交互有音效反馈。
- 默认音量克制，不盖过视频体验。
- 可通过全局开关临时关闭音效，避免答辩环境尴尬。

## 11. 答辩表达重点

### 11.1 技术选型

不要只说“用了特效”。建议表达为：

> 我们把互动特效分成高频轻量反馈、中频情绪表达、低频剧情爆发三个层级。高频特效用 Flutter 代码动画保证性能，中频特效用 sprite 粒子提升观感，低频关键镜头用 shader 做全屏冲击，兼顾工程可控和视觉表现。

### 11.2 创新点

- 用户不是被动看短剧，而是在剧情高光点参与情绪表达和路线选择。
- 特效不是单个动画贴片，而是视频层、粒子层、shader 层、反馈层的组合。
- 后端缺失时仍能用本地 mock 数据验证完整互动链路。

### 11.3 合规说明

- 纯代码和 shader 动效无外部素材版权风险。
- 粒子 sprite 优先使用 AI 生成，并记录生成工具、日期、用途和核心 prompt。
- 外部下载粒子/音效只使用许可证明确的素材。
- 外部下载素材记录来源、作者、许可证和下载日期。

## 12. 风险与应对

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| shader 在低端设备上掉帧 | 影响录屏和演示稳定性 | 控制持续时间，降低采样复杂度，保留非 shader fallback |
| 外部素材 license 不清楚 | 影响公开仓库合规 | 粒子优先 AI 生成；外部素材只使用 CC0 或明确可商用素材，不确定则不用 |
| Rive 学习成本超预期 | 拖慢主线 | 先用纯 Flutter 方案，Rive 作为增强项 |
| 音效干扰视频体验 | 答辩现场观感差 | 音量克制，并提供关闭开关 |
| 包依赖引入过多 | 构建和维护复杂 | 第一轮只加 `flutter_animate` 和 `confetti` |

## 13. 最小可交付版本

如果时间压缩，只保留以下内容：

1. `flutter_animate + confetti` 增强现有 6 种特效。
2. 1 个 shader 剧情爆发。
3. 3-5 个 AI 生成 sprite 粒子。
4. `docs/ASSET_LICENSES.md`。
5. 一段 60-90 秒录屏展示：

   - 双击爱心

这就是本轮特效升级的最低成功线。
